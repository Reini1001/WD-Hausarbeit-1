Animation

Animating with SVGs is similar to animating in CSS. You can, in fact, even animate some SVG-properties with CSS, though we won’t be taking a look at that, since it's the same as any other CSS animation. In this tutorial, I’ll try to showcase some of the most important functions when animating with SVGs using these examples:

The Vanishing Square
Two Clockhands
A Moving Circle

It is however important to mention that Animation is a huge topic and that I cannot cover all of it. There will be some links in the afterword for you to get more information.

Example 1: Vanishing Square

Before we can animate a shape in SVG, we need to define it. In this case, it's a 180px square in the middle of the SVG. Note: you can look at the code by clicking the button below.

Next, you need to define an animate-tag. I usually do this inside the tag of the shape I’m animating (in this case the rect), but we’ll be looking at another way later.

We’re animating the opacity, which is a CSS-property, so we have to define the attributeType as CSS. The attributeName is always required and refers to what we’re animating (opacity in this case). values and keyTimes work very similar to the percentual key-frames in CSS. All it’s saying is, that we want the opacity to change from 1 to 0 and back to 1. dur is the duration that one cycle lasts. Finally, the repeatCount defines how often the animation plays (in this case an infinite amount of times).

Using CSS I simply defined that the square is filled with a variable called green, that its frame (stroke) is 3px wide and that said frame has the color of the variable darkest. 

            
HTML:
<svg xmlns="http://www.w3.org/2000/svg" id="square" width="200px" height="200px">
    <rect height="180px" width="180px" x="10px" y="10px">
        <animate attributeType="CSS"
                 attributeName="opacity"
                 values="1 ; 0 ; 1"
                 keyTimes="0 ; 0.5 ; 1"
                 dur="5s"
                 repeatCount="indefinite"/>
    </rect>
</svg>

CSS:
#square > rect {
    fill: var(--green);
    stroke: var(--darkest);
    stroke-width: 3px;
}

Example 2: Clockhands

The clock is just a big white circle with a black border, six lines that are rotated to fit the hour positions, another white circle to cut the lines off and a black circle as a connection point for the handles. One thing to note here is, that you can use % instead of px for almost every attribute, as shown in this example

The clock hands are lines that are rotating. This is done using animateTransform instead of a normal animate. The attributeName needs to be transform for this one and the attributeType is XML. The type is used to define which transformation we want to use (in this case, rotate).

The attributes from and to are used when you only want to define a starting and an ending point in your animation. Since we only want the clock hands to spin in a circle, defining them as 0 and 360 respectively works just fine. Finally, we only need a dur (5 seconds and 60 seconds respectively) and a repeatCount (indefinite).

Using CSS, we make sure to set the transform-origin to center. The transform-origin defines the point from which all transformations originate and resides in the top left corner of the SVG by default. Setting it to center for the lines makes sure that they rotate properly. 

            
HTML:
<svg xmlns="http://www.w3.org/2000/svg" id="clock" width="200px" height="200px">

    <!-- The clock -->

    <circle r="40%" cx="50%" cy="50%" fill="white" stroke-width="2%" stroke="black"></circle>
    <line x1="50%" y1="90%" x2="50%" y2="10%" stroke-width="1%" stroke="black"></line>
    <line x1="50%" y1="90%" x2="50%" y2="10%" stroke-width="1%" stroke="black" transform="rotate(30)"></line>
    <line x1="50%" y1="90%" x2="50%" y2="10%" stroke-width="1%" stroke="black" transform="rotate(60)"></line>
    <line x1="50%" y1="90%" x2="50%" y2="10%" stroke-width="1%" stroke="black" transform="rotate(90)"></line>
    <line x1="50%" y1="90%" x2="50%" y2="10%" stroke-width="1%" stroke="black" transform="rotate(120)"></line>
    <line x1="50%" y1="90%" x2="50%" y2="10%" stroke-width="1%" stroke="black" transform="rotate(150)"></line>
    <circle r="35%" cx="50%" cy="50%" fill="white" stroke-width="0"></circle>
    <circle r="4%" cx="50%" cy="50%" fill="black" stroke-width="0"></circle>

    <!-- The clock hands -->

    <line x1="50%" y1="50%" x2="50%" y2="18%" stroke-width="2%" stroke="black">
        <animateTransform attributeName="transform"
                          attributeType="XML"
                          type="rotate"
                          from="0"
                          to="360"
                          dur="5s"
                          repeatCount="indefinite"/>
    </line>
    <line x1="50%" y1="50%" x2="50%" y2="30%" stroke-width="3%" stroke="black">
        <animateTransform attributeName="transform"
                          attributeType="XML"
                          type="rotate"
                          from="0"
                          to="360"
                          dur="60s"
                          repeatCount="indefinite"/>
    </line>
</svg>

CSS:
#clock > line {
    transform-origin: center;
}

Example 3: Moving Circle

text here

            
HTML:
<svg xmlns="http://www.w3.org/2000/svg" id="circle" width="200px" height="80px">
    <circle id="moving_circle" r="20px" cx="25px" cy="40px"/>

    <!-- The animation is linked with an xlink instead of being nested in the circle -->

    <animate xlink:href="#moving_circle"
             attributeName="cx"
             from="25"
             to="400"
             dur="3.5s"
             repeatCount="1"
             begin="click"
             restart="whenNotActive"/>
</svg>

CSS:
#circle {
    background-color: #FFFFFF4F;
    border: 3px solid var(--darkest);
    border-radius: 1rem;
    margin-top: 1rem;
}

#circle > #moving_circle {
    fill: var(--red);
    stroke: var(--darkest);
    stroke-width: 3px;
}
